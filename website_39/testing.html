Testing

Testing Strategy

To ensure that the key functionalities work perfectly we used Test-Driven-Development (TDD) which was the most recommendable software development process. Therefore, our testing process is comprised of Unit & Integration Testing, Compatibility Testing, Responsive Design Testing, Security Testing, Automated Testing and User Acceptance Testing.

Unit and Integration Testing

We divided the project into several pieces. Each of our developers works on separate functions and features and finally assemble and integrate all pieces together. So, we implement test-driven development. We did unit and integration testing during developing the App.

Unit testing makes sure each fundamental function in the App behaves correctly, so errors will not accumulate. Also, if the App goes wrong, we can easily and fast identify the bug. This is especially useful when the team tries to integrate all features together. After putting everything together, we did integration testing to ensure the whole system acts properly.

We use localhost port 8100 as a server host to develop and test the basic functions.

Unit test on APIs.

When using google map API, all requests sent from localhost are denied by Google API server. To allow testing on the correctness of the result of the API return result, we added an extension on the browser. All of the team members are using Chrome as a testing tool, so we add the Access-Control-Allow-Origin[2] plugin in Chrome. Which can automatically add to response header rule - 'Access-Control-Allow-Origin: *' when making an API request.

Integration testing on App

We did integration testing in two steps. Firstly, we used ionic devapp to have a genal test because it is a fast way. After we implement all key features, we use Xcode to test directly on target devices. We use ionic devapp as main integration testing tool.

Performance

Speech recognition accuracy testing

For each test case, speak 10 sentences with 121 words. The accuracy is calculated by the total number of words correctly recognised / total words. Before the test, we spoke close to the microphone in a very quiet place and selected sentences and words so that the accuracy is 100%.

Language understanding accuracy testing

We prepared 68 sentences. The accuracy is calculated by the total number of sentences comprehended correctly/ total sentences. During the test, we spoke close to the microphone in a very quiet place and make sure the accuracy of speech recognition is 100% so that the uncertainty of speech recognition will not affect this test.

After testing, the got the overall accuracy is 69%, which is not very reliable. By analysing our test cases, we found that when the sentence is short and simple with a clear grammar without too many unnecessary words, the performance will be much better.

Automated Testing

With automated testing, that time is cut drastically. The work for automated testers is instead spent coding the tests and making improvements to these tests repeatedly as adjustments are needed. Once the test is complete, however, automated testing allows for the recycled use of tests so that they do not have to go through this whole process again.[3]

We used automated testing in two parts, language understanding API calls and distance measurement. For both parts, we wrote two separate testing scripts.

User Acceptance Testing

After we finalise our app, we invited some users to do some test and complete a survey. We asked users to complete the following tasks:

1.Without any hint, try to use the app to get information about your surrounding building.

2.Use the voice assistance only to interact with the app at different places.

3.Try to use the app without looking at the screen

4.Can you predict the functions of these buttons?

The survey contains the following questions:

1.Happy with the UI?

2.How long did you finish the first task?

3.Is Voice assistance easy to use? Please rank out of 10.

4.Do you satisfy with the recommendation result?

5.How do you feel about the app? Rank.

6.Any suggestions?

We invite 12 users among our classmates, and we ask them to complete the tasks and survey questions abouve. The result indicates that, most of users (11/12) are satisfied with our UI, and half of them mentioned that the UI is clean and simple. Half of them (6/12) saied that they are happy to use the voice assistance instead of clicking buttons. 8 of them can predict functions of all buttons without preknowledge. After some tries, all of them can use voice assistance function without looking at the screen. But only 2 of them are satisfied with the result of recommendation. All the users are happy with the app, the average marks they gave for this app is 7.8 out of 10.
